What does fast-forward in Git mean?
In Git, fast forward means when you update branch that is ahead of your checked-out one. 

Why should you have different branches?
Multiple branch helps each developer to isolate their own work and it helps to keep track of individual commits. 

In a project how many remote Git repositories do you usually have?
You can have more than one remote Git repositories. But in single project there is only one remote origin.
